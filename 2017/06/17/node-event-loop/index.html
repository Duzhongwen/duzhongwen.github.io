简单谈谈node异步IO与事件循环

随着JavaScript生态的迅猛发展，JS的活动范围已经不仅仅局限的浏览器前端，随着NodeJS的诞生已然再向服务端迅速的扩张。一些大公司也在开始启用Node来重构以前的业务功能。今天就简单谈谈nodeJS的2大核心特点：异步IO、事件循环。

异步IO

虽然NodeJS作为后端的运行平台，但它仍然保留了前端浏览器的一些特点和JS本身的语言特性。熟悉前端的工程师应该知道，浏览器的JS是以单线程的方式来运行的，虽然浏览器本身有维护其他线程，但我们所写的代码都是在同一条线程里面执行的，
这样的好处是程序员在编写代码的时候不用考虑多线程下的状态切换，也就不会出现死锁或者一些其他的IO操作冲突的问题，同时也降低了线程来回切换所造成的性能开销。当然NodeJS作为JS在服务端的运行环境，也延续了跟浏览器相同的特性，单
线程的运作模式。

![](http://img.blog.csdn.net/20161206201855134?watermark/2/text/aHR0cDovL2Jsb2cuY3Nkbi5uZXQvTmV4dFN0YW5k/font/5a6L5L2T/fontsize/400/fill/I0JBQkFCMA==/dissolve/70/gravity/SouthEast)

既然是NodeJS是单线程的，那你可能就有疑问了，那程序要如何处理并发的问题呢？这里就要引入异步IO的理念了。异步本来也不是一个比较新颖的概念，在其他语言里面早有应用，但是像Node这样如此频繁的使用异步IO操作还是仅此一家。现在我们
就简单讲一个读写文件的例子来理解异步IO，它与前端Ajax的调用方式是非常类似的：

执行之后我们可以看到，’发起读取文件‘是在’读取文件完成‘之前输出的。’读取文件完成‘所处的回调函数取决于读取文件何时完成。从这里可以看出在读取文件的时候程序是在继续往下执行的，当文件读取完成之后再通知程序执行读取成功的回调，读取文件的
IO过程程序并不是阻塞的，这就是异步IO的调用模式。

![](https://ss3.bdstatic.com/70cFv8Sh_Q1YnxGkpoWK1HF6hhy/it/u=826586400,1734338889&fm=11&gp=0.jpg)

为什么Node中异步调用如此频繁，其实这主要得益于Node的发布者在底层开发出大量基于C++的异步API可供开发者调用，包括文件读取、网络请求等等模块。这样的优势在于程序无需等待每一步IO操作完成才能执行下一步的动作，各I/O操作在底层并发完成，成
功后再通知主线程执行回调。这样可以大大的提升执行效率，不用浪费时间等待漫长的I/O调用结束。

事件循环

Node.js 的每一个 API 都是异步的，并作为一个独立线程运行，使用异步函数调用，并处理并发。那么异步操作完成之后主线程是如何知道并执行对应的回调的呢？这里就引入了事件循环的机制，Node.js 单线程类似进入一个while(true)的事件循环，通过它查
询事件观察者，直到没有事件观察者退出，每个异步事件都生成一个或者多个事件观察者，如果有事件发生就调用该回调函数，每执行一次循环体的过程被称为一次Tick。

![](https://timgsa.baidu.com/timg?image&quality=80&size=b9999_10000&sec=1497719641464&di=8fffc44d0f23746b02f17ffa661c4633&imgtype=0&src=http%3A%2F%2Fwww.th7.cn%2Fd%2Ffile%2Fp%2F2015%2F12%2F26%2Ff07849eb815386821a444d026dd75e7c.jpg)

在node进程启动时，底层C++源码会进入到main函数，同时会初始化一个数据结构default_loop_struct,这个数据结构是事件循环的核心，然后加载js文件并执行。当程序执行需要调用异步API的时，如读写文件，这些异步的API模块将会lib模块>C++模块>libuv
接口>最终系统底层的API>系统返回结果和JavaScript代码（callback函数的js代码）两部分，然后被封装成一个IO观察者并保存在default_loop_struct中。引入观察者的模式是比较常见的事件监听方式，主要是用于判断观察者中是否有需要处理的事件。事件循
环则是从观察者那里取出事件并执行。

进入事件循环，调用libuv的事件循环的入库函数uv_run()，如果存在异步的IO操作，那么查询default_loop_struct保存的对应的IO观察者，如果存在观察者中存在系统返回的数据，则执行对应callback的JS代码，执行完毕之后main函数继续调用libuv的事件循
环入库uv_run()，node进入下一个Tick。

有一点需要注意：只有用户的js代码全部执行完之后，node才会进入事件循环，并且uv_run()的while循环其实只做了一件事情就是判断default_loop_struct中是否还有IO观察者，如果没有则uv_run()退出，node进程也随之退出。若还有IO观察者，则进入epoll
_wait()，线程挂起，等待系统返回IO操作的结果再执行callback内容，没有数据到来则一直等待。

总结：事件循环 和 观察者是Node异步IO构成的重要核心。
